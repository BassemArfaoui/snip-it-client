How to properly handle async/await errors inside React's useEffect hook without triggering unhandled promise rejections?

Marcus Neo AUTHOR
Asked 5 hours ago
visibility 1,240
comment 2

I am building a dashboard widget that fetches data from an API when the component mounts. I'm using async/await syntax for cleaner code, but I'm struggling with error handling inside useEffect.

The linter warns me about not making the effect callback async, but when I define an inner async function, I sometimes get unhandled promise rejections if the component unmounts before the fetch completes.

Here is my current implementation:

useEffect(() => {
  const fetchData = async () => {
    try {
      const response = await api.get('/widgets/stats');
      setData(response.data);
    } catch (err) {
      setError(err.message);
    }
  };
  fetchData();
}, []);

Is there a standard pattern or best practice to handle this gracefully, including cancellation support?

You should use an AbortController to handle the cleanup properly. This is the standard way to cancel fetch requests in useEffect when the component unmounts.

Here's how you can refactor your code:

useEffect(() => {
  const controller = new AbortController();
  const fetchData = async () => {
    try {
      const response = await api.get('/widgets/stats', {
        signal: controller.signal
      });
      setData(response.data);
    } catch (err) {
      if (err.name !== 'CanceledError') {
        setError(err.message);
      }
    }
  };
  fetchData();
  return () => controller.abort();
}, []);
